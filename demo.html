KINECT

KINECT

OVERVIEW

The Microsoft Kinect sensor is a peripheral device that functions much like a webcam. However, in addition to providing an RGB image, it also provides a depth map. Meaning for every pixel seen by the sensor, the Kinect measures distance from the sensor.

HOW DOES KINECT WORKS?

The Kinect sensor itself only measures color and depth.

Once that information is on your computer, lots more can be done like “skeleton” tracking.

PROCESSING

PROCESSING

OVERVIEW

Processing is an open source programming language and environment for people who want to create images, animations, and interactions.

START WITH INECT IN PROCESSING

The easiest way to install the library is with the Processing Contributions Manager Sketch → Import Libraries → Add library and search for “Kinect”. A button will appear labeled “install”.
Then, restart Processing.

## CODE First thing is to include the proper import statements at the top of your code. ```
import org.openkinect.processing.*;
```
## CODE As well as a reference to a Kinect object, i.e. ```
Kinect kinect;
```
## CODE Then in setup() you can initialize that kinect object: ```
void setup() { kinect = new Kinect(this); kinect.initDevice(); }
```
## CODE If you are using a Kinect v2, use a Kinect2 object instead. ```
Kinect2 kinect2; void setup() { kinect2 = new Kinect2(this); kinect2.initDevice(); }
```
## CODE Once you’ve done this you can begin to access data from the kinect sensor. ```
PImage (RGB) from the kinect video camera. PImage (grayscale) from the kinect IR camera. PImage (grayscale) with each pixel’s brightness mapped to depth (brighter = closer). PImage (RGB) with each pixel’s hue mapped to depth. int[] array with raw depth data (11 bit numbers between 0 and 2048).
```
## CODE If you want to use the Kinect just like a regular old webcam, you can access the video image as a PImage! ```
PImage img = kinect.getVideoImage(); image(img, 0, 0);
```
## CODE If you want the IR image: ```
void videoEvent(Kinect k) { // There has been a video event! }
```
## CODE If you want the IR image: ```
kinect.enableIR(true);
```
## CODE With kinect v1 cannot get both the video image and the IR image. They are both passed back via getVideoImage() so whichever one was most recently enabled is the one you will get. However, with the Kinect v2, they are both available as separate methods: ```
PImage video = kinect2.getVideoImage(); PImage ir = kinect2.getIrImage();
```
## CODE If you want the depth image, you can request the grayscale image: ```
PImage img = kinect.getDepthImage(); image(img, 0, 0);
```
## CODE As well as the raw depth data: For the kinect v1, the raw depth values range between 0 and 2048, for the kinect v2 the range is between 0 and 4500. ```
int[] depth = kinect.getRawDepth();
```
## CODE For the color depth image, use kinect.enableColorDepth(true);. And just like with the video image, there’s a depth event you can access if necessary. ```
void depthEvent(Kinect k) { // There has been a depth event! }
```
## CODE Unfortunately, b/c the RGB camera and the IR camera are not physically located in the same spot, there is a stereo vision problem. The Kinect v2 offers what’s called a “registered” image which aligns all the depth values with the RGB camera ones. This can be accessed as follows: ```
PImage img = kinect2.getRegisteredImage()
```
## CODE Finally, for kinect v1 (but not v2), you can also adjust the camera angle with the setTilt() method. ```
float angle = kinect.getTilt(); angle = angle + 1; kinect.setTilt(angle);
```
## CODE Here are all the useful functions you might need to use the Processing kinect library: ```
initDevice() — start everything (video, depth, IR) activateDevice(int) - activate a specific device when multiple devices are connect initVideo() — start video only enableIR(boolean) — turn on or off the IR camera image (v1 only) initDepth() — start depth only enableColorDepth(boolean) — turn on or off the depth values as color image enableMirror(boolean) — mirror the image and depth data (v1 only) PImage getVideoImage() — grab the RGB (or IR for v1) video image PImage getIrImage() — grab the IR image (v2 only) PImage getDepthImage() — grab the depth map image PImage getRegisteredImage() — grab the registered depth image (v2 only) int[] getRawDepth() — grab the raw depth data float getTilt() — get the current sensor angle (between 0 and 30 degrees) (v1 only) setTilt(float) — adjust the sensor angle (between 0 and 30 degrees) (v1 only)
```
